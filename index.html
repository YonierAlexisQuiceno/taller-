Funciones y Procedimientos

Un subprograma es como un "mini-programa" dentro de tu código principal. Siguiendo la Metáfora de la Fábrica, dividimos el trabajo en especialistas para mejorar la reutilización, la legibilidad y el mantenimiento del código.

La Acción (Procedimiento)

Orden directa para ejecutar una tarea. NO devuelve una respuesta utilizable. En JS no usamos void explícitamente, pero el concepto es el mismo.

function saludar() {
  console.log("¡Hola, clase!");
}

El Cálculo (Función)

Realiza una operación y obligatoriamente DEVUELVE un resultado útil a través de la palabra reservada return.

function sumar(a, b) {
  return a + b;
}

// Almacenamos el retorno:
let resultado = sumar(5, 2);

Parámetros vs Argumentos

1. El Parámetro (La Receta)

Es la variable definida en la declaración. Es el "espacio reservado".

function cocinar(ingrediente)
2. El Argumento (El Invitado)

Es el valor real enviado al momento de invocar (llamar) la función.

cocinar("Pollo");
El Secreto de C y TypeScript: En lenguajes tipados estrictos, la diferencia entre Función y Procedimiento es aún más clara, ya que te obligan a colocar la palabra void si no hay retorno, o el tipo de dato (ej. int) si sí lo hay.

Revelar Pista

En lenguajes como JavaScript, si una función termina su ejecución y no devuelve nada explícitamente, por defecto devuelve el valor undefined (indefinido).

¿Dónde vive mi variable?

En JavaScript, las variables declaradas fuera de cualquier función son Globales. Las declaradas con let o const dentro de las llaves { } de una función o un bloque son Locales / de Bloque y se destruyen al terminar. 

// VARIABLE GLOBAL
let saldoBancario = 1000;

function intentarModificar() {
  // VARIABLE LOCAL
  let bono = 500;
  saldoBancario = saldoBancario + bono;

  // ALCANCE DE BLOQUE
  if (true) {
    let bloque = "Solo visible aquí";
  }
}

intentarModificar();
console.log("Saldo:", saldoBancario);
console.log(bono); // ¡ERROR!

Paso por Valor (La Fotocopia)

En JavaScript, cuando pasamos un dato primitivo (como un Number, String o Boolean) a una función, el programa hace una copia. Si la función modifica el parámetro, solo está rayando la fotocopia. ¡El original está a salvo! 

// Parámetro 'ingrediente' es una COPIA
function cocinar(ingrediente) {
  ingrediente = 2; // 2 representa "Cocinado"
}

let miCena = 1; // 1 representa "Crudo"

cocinar(miCena); // Enviamos el argumento

console.log("Estado:", miCena);
// ¡Sigue imprimiendo 1 (Crudo)!

I Ámbito de Variables (Scope) y Memoria

Fundamento: El ámbito determina la visibilidad y el ciclo de vida de una variable en la memoria (Stack). Las variables globales residen en memoria durante toda la ejecución, lo que representa un riesgo de seguridad y dificulta la depuración. Las variables locales nacen al invocar la función y se destruyen al finalizar (return), garantizando el encapsulamiento de los datos.

En el siguiente código, observaremos una variable global (impuestoGlobal), una variable local (costoLogistica) y una variable de bloque (descuentoEspecial), la cual solo existe dentro de la estructura condicional if.

// 1. Variable Global let impuestoGlobal = 0.19; function calcularPrecioFinal(precioBase) { // 2. Variable Local let costoLogistica = 5000; let historial = "Precio base: $" + precioBase + "\n"; if (precioBase > 100000) { // 3. Variable de Bloque (Solo existe en este if) let descuentoEspecial = 0.10; let valorDescuento = precioBase * descuentoEspecial; precioBase = precioBase - valorDescuento; historial += "Descuento aplicado: -$" + valorDescuento + "\n"; } let total = precioBase + (precioBase * impuestoGlobal) + costoLogistica; historial += "Total Final: $" + total; return historial; }

II Funciones vs. Procedimientos

Fundamento: La principal diferencia radica en el flujo de la información. Un procedimiento (equivalente a void en lenguajes estrictos) ejecuta una acción, como imprimir en pantalla o modificar la interfaz, pero no devuelve información al programa principal. Una función procesa datos y obligatoriamente utiliza la instrucción return para devolver un resultado, el cual debe ser capturado por una variable en la ejecución principal.

// Procedimiento (No retorna valores, solo ejecuta acciones) function registrarLog(mensaje, idSalida) { let fecha = new Date().toLocaleTimeString(); let logFormateado = `[SISTEMA ${fecha}]: ${mensaje}`; document.getElementById(idSalida).innerText = logFormateado; } // Función (Retorna un valor calculado) function calcularRendimiento(capital, tasa, tiempo) { let beneficio = (capital * tasa) * tiempo; return beneficio; // Obligatorio retornar el dato } // Ejecución Principal let inversion = 5000; let resultadoFinal = calcularRendimiento(inversion, 0.05, 3); // Captura del retorno registrarLog("Rendimiento calculado: $" + resultadoFinal, 'out-eje2');

III Gestión de Parámetros (Valor vs. Referencia)

Fundamento: Cuando enviamos un dato primitivo (como un número o texto) a un subprograma, ocurre un Paso por Valor; se envía una "fotocopia" del dato, protegiendo la variable original de modificaciones internas. Por el contrario, al enviar estructuras complejas (como un Objeto o un Arreglo), ocurre un Paso por Referencia; se envía la dirección de memoria exacta, por lo que cualquier alteración en el subprograma afectará los datos globales.

// Recibe un tipo primitivo (Paso por Valor) function intentarCobrarPenalidad(saldoLocal) { saldoLocal = saldoLocal - 100; // Altera la copia, no el original } // Recibe un objeto (Paso por Referencia) function actualizarInventario(baseDatos) { baseDatos.stock = baseDatos.stock - 1; // Altera el objeto original en memoria } // --- Ejecución Principal --- let miSaldoBancario = 1000; intentarCobrarPenalidad(miSaldoBancario); // Resultado: miSaldoBancario sigue siendo 1000. let producto = { nombre: "Laptop", stock: 50 }; actualizarInventario(producto); // Resultado: producto.stock ahora es 49.

IV Proyecto de Integración: Sistema de Facturación

Fundamento: Este sistema requiere orquestar múltiples subprogramas. Se define un flujo donde la salida (return) de una función se convierte en el argumento de entrada de la siguiente, finalizando con un procedimiento que formatea la salida visual. Se demuestra la cohesión modular y el correcto tránsito de datos sin depender de variables globales innecesarias.

const IVA = 0.19; // Única variable global justificada function aplicarDescuento(subtotal) { return subtotal - (subtotal * 0.10); // Descuento local del 10% } function calcularTotal(valorDescontado) { return valorDescontado + (valorDescontado * IVA); } function mostrarFactura(cliente, total, idSalida) { // Procedimiento de salida formateada let texto = `=== FACTURA ===\nCliente: ${cliente}\nTotal: $${total}\n===============`; document.getElementById(idSalida).innerText = texto; } // Ejecución Orquestada let subtotalInicial = 200000; let subtotalDescuento = aplicarDescuento(subtotalInicial); let granTotal = calcularTotal(subtotalDescuento); mostrarFactura("Corporación Omega", granTotal, 'out-proyecto');
